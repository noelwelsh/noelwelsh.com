Collections

Collection classes allow us to store lots of data, and are particularly useful when we don’t know how much data we’ll need to hold. For example, we could use a collection class to hold a customer’s orders. They may have never ordered from us, in which case they have no orders, or they might be our best customer and have made thousands of orders.

Different collection classes will hold data in different ways, which changes how quickly they can perform different operations. Scala comes with an extensive library of collections that allow us to choose appropriate tradeoffs. For example, we can quickly add new elements to a List but cannot quickly find an element in a List. An ArrayBuffer can quickly find elements but adding new elements may take some time.

In this section we’re going to look at two key categories in Scala’s collection library: sequences and maps.

We will start with sequences. We begin with basic operations on sequences, and then briefly examine the distinction Scala makes between interface and implementation, and mutable and immutable sequences. We then explore in depth the methods Scala provides to transform sequences.

Next we will cover the other main categories of collection classes: maps and sets. We will discover that they share a great deal in common with sequences, so most of our knowledge transfers directly.

We finish with discussion of ranges, which can represent large sequences of integers without storing every intermediate value in memory.

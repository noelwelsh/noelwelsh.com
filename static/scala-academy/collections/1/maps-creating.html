Creating Maps

The way we create maps is very similar to the way we create sequences. The most important difference is that maps have both keys and values associated with those keys. This means we have to specify both when we create a map, and maps have two type parameters: one for the key type and one for the value type.

Let’s see some examples. First the vararg constructor.

val letters = Map("a" -> 1, "b" -> 2, "c" -> 3)
// letters: Map[String, Int]

This works the same as we’ve seen with sequences, but we specify the keys (the Strings) and the values (the Ints), separating the key and value with →.

We haven’t seen the → syntax before. It creates a tuple. A tuple is a collection with fixed size and the elements may vary in type. This is in contrast to a sequence, which has a varying size but the elements must have same type. We’ll learn more about tuples in another section.

In the next example we construct an empty map.

val empty = Map.empty[Int,Boolean]
// empty: Map[Int, Boolean]

Note that here we specify both the type of the keys (Int) and the type of values (Boolean).

We can construct a new map by appending two existing maps using the method ++. The new map will contain the keys and values from the existing maps, with values from the right-hand map overriding values from the left-hand map when both contain the same key. Notice in the example below both maps contain the key "c", and the result of appending them takes the value from the map right.

val left = Map("a" -> true, "b" -> false, "c" -> false)
// left: Map[String,Boolean] = Map(a -> true, b -> false, c -> false)

val right = Map("c" -> true, "d" -> false)
// right: Map[String,Boolean] = Map(c -> true, d -> false)

left ++ right
// Map[String,Boolean] = Map(a -> true, b -> false, c -> true, d -> false)
